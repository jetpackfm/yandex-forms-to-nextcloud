## О проекте

Автоматизация сбора ответов из заполненных форм в таблицу локального Nextcloud.

## Задача

Коллега на выставках собирала данные на бумажные носители, после чего переносила собранную информацию вручную в Excel таблицы. Возникла рабочая задача: перевести коллегу с бумажных анкет на электронные и автоматизировать сбор данных.

## Решение

Я выбрал следующую схему. Для создания форм остановился на сервисе Яндекс Формы. При заполнении и отправке самой формы используется интеграция внутри сервиса через API и метод запроса POST, которая посылает данные в формате JSON на платформу n8n, где они обрабатываются и передаются в подготовленную таблицу Nextcloud.

## Архитектура

Внешний мир: Яндекс.Формы → HTTPS → VPS (Caddy:8443)  
Внутренний контур: VPS → WireGuard → Виртуалка (n8n:5678) → Nextcloud Tables  
Администрирование: ПК → SSH-туннель → Виртуалка (n8n:5678)  

- Публичный эндпоинт: `https://n8n.jetpack-jobit.ru:8443/webhook/yandex-form`
- Все компоненты, кроме вебхука, изолированы от внешней сети
- Администрирование n8n — через SSH-туннель или WireGuard VPN

## Технологии

- **WireGuard + AmneziaWG** — туннель VPS ↔ домашний сервер, обфускация
- **VPS** — публичный шлюз, реверс-прокси
- **Caddy** — автоматический SSL и маршрутизация запросов
- **n8n** — обработка вебхуков и логика автоматизации
- **Nextcloud** — хранение данных в таблицах

**Проект основывается на предыдущих проектах:**

- [Сборка и настройка домашнего сервера](https://github.com/jetpackfm/mpd-server)  
- [Удаленный доступ к Nextcloud через WireGuard и AmneziaWG](https://github.com/jetpackfm/nextcloud-family-vpn)

## Безопасность

- HTTPS (Let's Encrypt) на всех внешних соединениях
- Порт 8444 (админка n8n) доступен только через защищённый туннель
- VPS: iptables с политикой DROP по умолчанию
- WireGuard для связи VPS ↔ домашний сервер
- AmneziaWG для клиентского доступа (обфускация)
- Данные форм передаются только по защищённым каналам

## Жизненный цикл заявки

1. Ответственный копирует ссылку на форму и добавляет `?expo=Название`
2. Генерирует QR-код, распечатывает, несёт на мероприятие
3. Участник сканирует QR, заполняет форму, нажимает "Отправить"
4. Яндекс.Формы отправляют JSON на n8n
5. n8n парсит данные и записывает в Nextcloud Tables
6. Ответственный видит новую запись в таблице

> [!NOTE]
> Проект реализован на домашней инфраструктуре с двойным NAT и CGNAT,
> что наложило дополнительные требования к сетевой безопасности.
> При переносе в продакшен с публичным сервером архитектура упрощается:
> Яндекс.Формы → n8n (публичный) → Nextcloud (публичный), без VPS-прокладки
> и WireGuard-туннеля. Однако использованные практики безопасности
> (iptables, reverse proxy, автовыпуск сертификатов) применимы в любой среде.
